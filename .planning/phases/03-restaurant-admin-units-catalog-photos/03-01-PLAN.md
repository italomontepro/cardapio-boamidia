<task id="03-01-T02" type="auto">
  <title>Add Drizzle relations() to schema.ts and create verify-catalog.ts stub</title>
  <read_first>
    - src/db/schema.ts (must read entire file — append relations after the existing table definitions; do NOT alter any pgTable definition)
    - src/db/index.ts (confirms drizzle is instantiated with { schema } — required for db.query relational API)

  </read_first>
  <action>
    1. In `src/db/schema.ts`, add this import near the top (alongside the existing drizzle-orm/pg-core import):
       ```typescript
       import { relations } from 'drizzle-orm'
       ```
       Then APPEND (do not modify existing tables) at the end of the file:
       ```typescript
       export const restaurantsRelations = relations(restaurants, ({ many }) => ({
         units: many(units),
         categories: many(categories),
         products: many(products),
       }))

       export const unitsRelations = relations(units, ({ one, many }) => ({
         restaurant: one(restaurants, { fields: [units.restaurantId], references: [restaurants.id] }),
         productAvailability: many(productAvailability),
       }))

       export const categoriesRelations = relations(categories, ({ one, many }) => ({
         restaurant: one(restaurants, { fields: [categories.restaurantId], references: [restaurants.id] }),
         products: many(products),
       }))

       export const productsRelations = relations(products, ({ one, many }) => ({
         restaurant: one(restaurants, { fields: [products.restaurantId], references: [restaurants.id] }),
         category: one(categories, { fields: [products.categoryId], references: [categories.id] }),
         productAvailability: many(productAvailability),
       }))

       export const productAvailabilityRelations = relations(productAvailability, ({ one }) => ({
         product: one(products, { fields: [productAvailability.productId], references: [products.id] }),
         unit: one(units, { fields: [productAvailability.unitId], references: [units.id] }),
       }))
       ```
       NO migration is needed — relations() is Drizzle metadata only, not SQL.
    2. Create `scripts/verify-catalog.ts` as a STUB mirroring `scripts/verify-restaurants.ts` structure exactly:
       - Same dotenv `.env` walk-up loader.
       - Same `ws` import for the realtime shim.
       - Read `url`, `secretKey` from env.
       - `main()` that: dynamically imports `{ db }` from `'../src/db'` and `{ restaurants, units, categories, products }` from `'../src/db/schema'` and `{ eq, asc }` from `'drizzle-orm'` AFTER `config()`.
       - SMOKE CHECK: `const allUnits = await db.select().from(units)` then `console.log(\`SMOKE PASS: units table reachable, ${allUnits.length} rows\`)`.
       - Pick a test restaurant: `const [seedRestaurant] = await db.select().from(restaurants).limit(1)` — if none, `throw new Error('verify-catalog: no restaurant seeded — run npm run seed first')`. Store `const restaurantId = seedRestaurant.id` for later plans to use.
       - Also test the relational query works: `const cats = await db.query.categories.findMany({ where: eq(categories.restaurantId, restaurantId), with: { products: true } })` then `console.log(\`RELATIONS PASS: relational query returned ${cats.length} categories\`)`.
       - End with `console.log('ALL CHECKS PASSED')`.
       - Final line: `main().then(() => process.exit(0)).catch((e) => { console.error('FAIL:', e.message); process.exit(1) })`.
       - Add comment header noting later plans (03-02, 03-03, 03-04) extend this script with per-requirement assertions.
    3. Add to `package.json` scripts: `"verify-catalog": "tsx scripts/verify-catalog.ts"`.
  </action>
