# Phase 04.1 Plan 04: Unit Wizard UI + Map Step + "Sem localização" Badge Summary

**Refactored the single-step unit Dialog into a 3-step wizard (Básico → Contato/Horário → Localização) with an embedded Leaflet map that auto-geocodes the typed address and supports manual search/drag-to-adjust, plus a discreet badge flagging units without coordinates.**

## Accomplishments
- `page.tsx`'s `UnitRow` now carries `lat: number | null` / `lng: number | null`, and the Supabase query selects `lat, lng` alongside the existing columns, with defensive `Number()` coercion in case PostgREST ever returns numeric columns as strings
- `unit-table.tsx` renders a discreet `Badge variant="outline"` reading "Sem localização" next to the unit name whenever `lat`/`lng` is null — no new table column, keeping the table compact (D-06)
- `unit-form-dialog.tsx` rewritten as a 3-step wizard inside the same Dialog and same `useForm<UpsertUnitInput>` + `zodResolver(upsertUnitSchema)` instance: Step 1 (Nome), Step 2 (WhatsApp/Endereço/Horários), Step 3 (Localização/map) — fields stay mounted (display:none toggling) so RHF registration and user input are never lost when navigating back and forth
- Progress indicator "Etapa N de 3" plus 3 dots, and Próximo/Voltar navigation; `goNext()` calls `form.trigger(STEP_FIELDS[step])` so the admin cannot advance past a step with invalid required fields, while Step 3 has no required fields — location stays fully optional (D-07)
- Step 3 embeds `UnitLocationMap` (imported only from the SSR-safe `unit-location-map-loader`), auto-geocodes the Step-2 address exactly once on Step-3 entry via a `useRef` guard, exposes a manual "Buscar" button + status text for re-geocoding, and a draggable pin whose `onChange` writes straight into the hidden `lat`/`lng` RHF fields — defaulting the map view to Brazil's geographic center (`-14.235, -51.9253`) when no coordinates exist yet
- `npm run build` (Turbopack) completed successfully with zero SSR `window`/`document` errors, confirming the dynamic-import isolation from Plan 03 holds end-to-end inside the wizard
