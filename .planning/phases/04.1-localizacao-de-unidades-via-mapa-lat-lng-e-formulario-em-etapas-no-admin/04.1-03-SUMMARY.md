# Phase 04.1 Plan 03: Leaflet Map Widget (Client-Only, Draggable Marker) Summary

**Standalone client-only Leaflet map widget (MapContainer + draggable Marker + Recenter) wrapped in a next/dynamic(ssr:false) loader, the sole import surface Plan 04's wizard will use.**

## Accomplishments
- `unit-location-map.tsx` created as a `'use client'` component rendering an interactive OpenStreetMap (Leaflet `MapContainer`/`TileLayer`) with a draggable `Marker` that reports new coordinates to the parent via `onChange` on `dragend`
- Default Leaflet marker icon paths fixed at module scope (`L.Icon.Default.mergeOptions`) using a bundler-agnostic `(png as {src?}).src ?? png` guard, preventing the broken-image-pin issue under Turbopack
- `Recenter` helper component (using `useMap()` + `map.setView`) added so the map pans to new lat/lng whenever props change post-mount — required because `MapContainer`'s `center` prop is read only once at mount
- `unit-location-map-loader.tsx` created as the `next/dynamic({ ssr: false })` wrapper exporting `UnitLocationMap`, with a "Carregando mapa…" loading fallback — this is the only module future consumers (Plan 04) should import, keeping Leaflet's `window`/`document` access out of the SSR tree
- `npm run build` (Turbopack) completed successfully with zero `window is not defined` / `document is not defined` errors, confirming Pitfall 2 is avoided
