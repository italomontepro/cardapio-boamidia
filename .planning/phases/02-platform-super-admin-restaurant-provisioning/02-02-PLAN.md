Include these helpers in the file:
```typescript
import { randomBytes } from 'crypto'

function generateTempPassword(): string {
  // Cryptographically secure. Mix classes to satisfy any Supabase password policy.
  return `${randomBytes(6).toString('hex')}Aa1!`
}

function isUniqueViolation(err: unknown): boolean {
  return (
    typeof err === 'object' && err !== null && 'code' in err &&
    (err as { code: string }).code === '23505'
  )
}

class ProvisioningError extends Error {
  constructor(public code: string, message?: string) { super(message); this.name = 'ProvisioningError' }
}
```

**createRestaurant(input: CreateRestaurantInput)**:
1. `const parsed = createRestaurantSchema.safeParse(input)`; on failure return `{ error: parsed.error.flatten().fieldErrors }`.
2. Pre-check slug uniqueness via `db.query.restaurants.findFirst({ where: eq(restaurants.slug, slug) })`; if found return `{ error: { slug: ['Este link já está em uso. Escolha outro slug para o restaurante.'] } }` (UX fast-path; the DB constraint below is authoritative).
3. `const tempPassword = generateTempPassword(); const supabaseAdmin = createAdminClient(); let createdAuthUserId: string | null = null;`
4. Wrap in try/catch around `db.transaction(async (tx) => { ... })`:
   - `const [restaurant] = await tx.insert(restaurants).values({ name, slug }).returning()`
   - `const { data: authData, error: authError } = await supabaseAdmin.auth.admin.createUser({ email: adminEmail, password: tempPassword, email_confirm: true })`
   - `if (authError || !authData.user) throw new ProvisioningError('admin_create_failed', authError?.message)`
   - `createdAuthUserId = authData.user.id`
   - `await tx.insert(adminUsers).values({ userId: authData.user.id, role: 'restaurant_admin', restaurantId: restaurant.id })`
   - `return { restaurant, authUser: authData.user }`
5. On success return `{ success: true, restaurant: { id, name, slug }, tempPassword, adminEmail }`.
6. In catch(err): if `createdAuthUserId` is set, `await supabaseAdmin.auth.admin.deleteUser(createdAuthUserId)` (compensation — Postgres rows already auto-rolled-back by the transaction). Then: if `err instanceof ProvisioningError && err.code === 'admin_create_failed'` return `{ error: { adminEmail: ['Não foi possível criar o usuário administrador. Verifique o e-mail.'] } }`; if `isUniqueViolation(err)` return `{ error: { slug: ['Este link já está em uso. Escolha outro slug para o restaurante.'] } }`; otherwise `return { error: { _form: ['Não foi possível salvar as alterações. Verifique os dados e tente novamente.'] } }`.

NEVER persist `tempPassword` anywhere (no DB column, no log line). It is returned only in the response object for one-time display (D-08).

**updateRestaurant(input: UpdateRestaurantInput)**:
1. `safeParse` with `updateRestaurantSchema`; on failure return `{ error: parsed.error.flatten().fieldErrors }`.
2. `try { await db.update(restaurants).set({ name, slug }).where(eq(restaurants.id, id)) } catch (err) { if (isUniqueViolation(err)) return { error: { slug: ['Este link já está em uso. Escolha outro slug para o restaurante.'] } }; return { error: { _form: ['Não foi possível salvar as alterações. Verifique os dados e tente novamente.'] } } }`
3. `revalidatePath('/admin')`; return `{ success: true }`.

**toggleRestaurantActive(id: string, isActive: boolean)**:
1. `await db.update(restaurants).set({ isActive }).where(eq(restaurants.id, id))`
2. `revalidatePath('/admin')`; return `{ success: true }`.

Import `revalidatePath` from `'next/cache'`. Use `db` (Drizzle runtime client) for ALL writes here — per RESEARCH Pitfall 3, the Drizzle runtime connection bypasses RLS (privileged Postgres role), exactly as proven by scripts/seed.ts. Do NOT use the @supabase/ssr client for writes.
