## Spacing Scale

Declared values (must be multiples of 4), consistent with existing code (`p-4`, `p-6`, `gap-3`, `gap-4`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inline padding |
| sm | 8px | Compact element spacing (form field gaps: `gap-2`) |
| md | 16px | Default element spacing, page padding (`p-4`), card content gaps |
| lg | 24px | Section padding, main content area (`p-6`, matches existing `(dashboard)/layout.tsx`) |
| xl | 32px | Layout gaps between major page sections |
| 2xl | 48px | Major section breaks (not expected in this phase's compact admin table layout) |
| 3xl | 64px | Page-level spacing (not expected in this phase) |

Exceptions:
- Table row height: minimum 44px (touch target for "Editar" / "Ativar/Desativar" action buttons on row), per accessibility default for tap targets.
- Dialog content padding: 24px (`lg`), consistent with shadcn `DialogContent` default.

## Typography

Derived from existing code patterns (`text-xl font-semibold` for page heading, `text-sm` for body/labels/descriptions) and shadcn `base-nova` defaults. Declares the 4-size/2-weight scale for this phase.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label | 14px (`text-sm`) | 500 (medium — shadcn `Label` default) | 1.5 |
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.2 |
| Display | 24px (`text-2xl`) | 600 (semibold) | 1.2 |

Notes:
- **Body/Label share 14px** — this matches existing login page (`text-sm` for inputs, labels, error messages) and the dashboard's empty-state copy.
- **Heading (20px/semibold)** is the page title role: "Painel da Plataforma — Restaurantes" (existing `<h1 className="text-xl font-semibold">`).
- **Display (24px/semibold)** is reserved for Dialog titles ("Novo Restaurante" / "Editar Restaurante") and the AlertDialog title ("Desativar restaurante?") — shadcn `DialogTitle`/`AlertDialogTitle` default to `text-lg`–`text-xl`; bump to 24px only if a Dialog title needs more visual weight than the page heading. **Default: use 20px (Heading) for Dialog/AlertDialog titles too**, reserving 24px Display only if the checker flags insufficient hierarchy. Two weights total: 400 (body/data cells) and 600 (headings/titles/CTAs).
- Table cell text: 14px/400 (Body role). Table header cell text: 14px/600 (re-uses Heading weight at Body size — acceptable since shadcn `TableHead` uses `font-medium`; round to 600 for the 2-weight contract).

## Color

This phase uses the existing neutral OKLCH token set from `src/app/globals.css` (Phase 1) — no new colors introduced. The "60/30/10" split maps onto existing tokens as follows:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` (`oklch(1 0 0)` — white) | Page background, dialog overlay backdrop area, main content surface |
| Secondary (30%) | `--card` / `--secondary` / `--muted` (`oklch(1 0 0)` / `oklch(0.97 0 0)`) | Table container (`Card` wrapping `Table`), table header row background (`--muted`), Dialog/AlertDialog surfaces (`--popover`) |
| Accent (10%) | `--primary` (`oklch(0.205 0 0)` — near-black) | **Reserved for:** primary CTA button ("Novo Restaurante" trigger, "Salvar" / "Criar Restaurante" submit buttons in Dialog), active/focused table row action button. Nothing else uses this color. |
| Destructive | `--destructive` (`oklch(0.577 0.245 27.325)` — red) | **Reserved for:** "Desativar" button/badge state, AlertDialog confirm button for deactivation (D-12), inactive-status `Badge` variant text/border accent. Never used for informational or decorative elements. |

Accent reserved for: the single primary CTA button per screen/dialog ("Novo Restaurante", "Criar Restaurante", "Salvar Alterações") and the active-state `Badge` (status = ativo, rendered with a green-tinted badge variant per D-01 — see note below).

**Note on D-01 "verde/cinza" badges:** CONTEXT.md specifies status badges as ativo=verde / inativo=cinza. The base-nova neutral theme has no green token defined in `globals.css`. For this phase, use:
- `inativo` → shadcn `Badge variant="secondary"` (uses `--secondary`, gray) — no new token needed.
- `ativo` → shadcn `Badge` with a Tailwind utility override `bg-green-100 text-green-800 dark:bg-green-900/30 dark:text-green-400` (or equivalent `border-green-600/30 text-green-700`). This is the **one approved exception** to the neutral-only palette, scoped strictly to the active-status badge — do not introduce green elsewhere in this phase.
